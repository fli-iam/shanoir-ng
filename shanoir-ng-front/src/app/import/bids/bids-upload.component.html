1. Select Bids archive

Please select a BIDS/.sef archive to import.

See BIDS specifications: https://bids-specification.readthedocs.io/en/latest/

MR data import

You can either import from DICOM data in sourceData (1) folder or directly from nifti in rawData (2).

Genaral archive organisation:

import.zip
- dataset_description.json
- README
- participants.tsv
- sourcedata
- - DICOM
- - - sub-subject_name
- - - - [ses-session-index] <= Not mandatory - - - - - shanoir-import.json
- - - - - modality[anat/func/dwi] - - - - - - [list of images as .DCM]
- rawData
- - sub-subject_name
- - - [ses-session-index] <= Not mandatory - - - - shanoir-import.json
- - - - modality[anat/func/dwi] - - - - - [list of images as 'name.nii' and possibly 'name.json']

(1) If sourceData/DICOM folders are found, rawData is ignored and data will be imported from DICOM.

(2) Otherwise, data will be imported from niftis in rawData.

Please notice that in both cases a specific file shanoir-import.json is mandatory to import data. (see further description)

dataset_description.json: https://bids-specification.readthedocs.io/en/latest/03-modality-agnostic-files.html#dataset-description

participants.tsv: a TSV/CSV, tab separated, with column headers: - participant_id (name - mandatory)
- subject_identifier
- sex (M/F)
- birthdate (yyyy/mm/dd)
- manual_hemispheric_dominance (LEFT/RIGHT/null)
- language_hemispheric_dominance (LEFT/RIGHT/null)
- imaged_object_category (PHANTOM/LIVING_HUMAN_BEING/HUMAN_CADAVER/ANATOMICAL_PIECE/LIVING_ANIMAL/ANIMAL_CADAVER - mandatory)

Non existing subjects will be created based on their name

shanoir-import.json is a json constructed to specify the studyID in which we import the data, the examination and the selected studycard.

{{'{'}}
   "studyId" : 10, => Study ID
   "examinationId": 0, => Examination ID (can be null or set to 0 to be created)
   "studyCardId": 1 => ID of the studyCard (must not be null)
{{'}'}}

If examinationId is set, it must be already existing in Shanoir
If not, it will be created dynamically Associated studycard must be existing too, and linked to studyId
